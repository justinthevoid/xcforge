## 1) Install xcforge

```bash
brew tap justinthevoid/tap && brew install xcforge
```

Alternative source build:

```bash
git clone https://github.com/justinthevoid/xcforge.git
cd xcforge && swift build -c release
sudo install -m 0755 .build/release/XCForgeCLI /usr/local/bin/xcforge
```

If your Homebrew prefix is `/opt/homebrew`, install to that path instead:

```bash
sudo install -m 0755 .build/release/XCForgeCLI /opt/homebrew/bin/xcforge
```

Use `which xcforge` after installation to confirm the path your MCP client should reference.

## 2) Verify your CLI is available

```bash
xcforge --help
```

You should see top-level workflow subcommands such as `build`, `test`, `sim`, `diagnose`, and `ui`.

## 3) Optional: set defaults for faster loops

```bash
xcforge defaults set --project MyApp.xcodeproj --scheme MyApp --simulator "iPhone 16 Pro"
xcforge defaults show
```

Defaults reduce repeated flags for build/test/simulator commands.

## 4) Run a first verification loop

```bash
xcforge build-test --json
xcforge test failures --include-console --json
```

If your run is UI-heavy, add simulator and evidence checks:

```bash
xcforge sim list --json
xcforge log start --mode smart --json
xcforge screenshot capture --json
xcforge log read --include app --include crashes --json
xcforge log stop --json
```

## 5) Configure MCP host integration

Follow [MCP Client Configuration](/docs/reference/mcp-client-configuration/) for client-specific JSON formats and file locations.

Quick start for Claude Code:

```bash
claude mcp add xcforge -- xcforge
```

## 6) Validate parity expectations

Before automating deeper workflows, review:

- [CLI Command Groups](/docs/guides/cli-command-groups/)
- [Workflow Patterns](/docs/guides/workflow-patterns/)
- [CLI and MCP Parity](/docs/reference/cli-mcp-parity/)

## 7) Troubleshooting baseline

If your first loop fails:

1. Confirm Xcode and simulator readiness.
2. Re-run `xcforge build --diagnose --json` for structured build errors.
3. Use `xcforge diagnose start` then `xcforge diagnose inspect` for correlated evidence.

If the run times out silently, xcforge captures a diagnostic snapshot automatically. The result message includes a `/tmp/xcf-diag-*.txt` path and a one-line summary of what xcodebuild was doing. For test suites that take longer than 3 minutes, pass `--long` to extend the timeout to 1800s.
